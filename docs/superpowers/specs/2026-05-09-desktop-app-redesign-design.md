# 桌面应用重构设计文档

## 概述

将 Claude Kanban 从 Web 应用（React + Express）重构为 Electron 桌面应用，重新设计 UI/UX，采用暖色调手工风格。

## 技术栈

| 层 | 技术 |
|---|------|
| 桌面壳 | Electron（主进程运行 Express 服务器 + SQLite） |
| 前端 | React 18 + TypeScript + Vite 5 |
| 样式 | Tailwind CSS 4.x |
| 服务端 | Express + WebSocket + better-sqlite3（现有代码复用） |

**架构变化：** Electron 主进程直接运行 Express 服务器，`better-sqlite3` 原生模块和 `child_process.spawn(claude)` 在 Node.js 环境中零改动复用。BrowserWindow 加载 React 前端，通过 `localhost:14567` API/WebSocket 通信。

## 设计语言：暖色调手工感

参考 Notion、Things 3 的设计气质——温暖、有质感、不冷冽。

### 字体

- UI 正文：系统无衬线字体栈 `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- 日志/代码：等宽字体 `'SF Mono', 'Fira Code', Menlo, Consolas, monospace`

### 圆角与阴影

- 卡片：`border-radius: 8px`，`border: 1.5px solid #e8d5c4`
- 按钮：`border-radius: 8px`
- 无大面积阴影，卡片靠边框区分，不使用浮起效果

### 状态配色

| 状态 | 英文 | 标签 | 图标 |
|------|------|------|------|
| 待启动 | pending | `📥` | 灰色/中性 |

| 待验收 | reviewing | `✅` | 蓝绿 |

## 页面结构

### 主窗口：左右分栏布局

```
┌──────────────────────────────────────────────┐
│  标题栏（可选，由 Electron 窗口装饰替代）      │
├──────┬───────────────────────────────────────┤
│      │  列表区              │  详情区         │
│ 左   │  ┌─────────────────┐ │ ┌─────────────┐│
│ 侧   │  │ 📁 项目A        │ │ │ 任务详情     ││
│ 导   │  │  ┌──────────┐   │ │ │ 描述         ││
│ 航   │  │  │ 任务卡片  │   │ │ │ 状态信息     ││
│      │  │  │ 任务卡片  │   │ │ │ 操作按钮     ││
│ 📥   │  │  └──────────┘   │ │ │ 日志区域     ││
│ ⚡   │  │ 📁 项目B        │ │ │             ││
│ ❓   │  │  ┌──────────┐   │ │ │             ││
│ ✅   │  │  │ 任务卡片  │   │ │ │             ││
│ 📦   │  │  └──────────┘   │ │ │             ││
│      │  └─────────────────┘ │ └─────────────┘│
│ 📁   │                       │               │
└──────┴───────────────────────┴───────────────┘
```

- **左侧导航栏**（固定宽度 ~56px）：图标按钮，垂直排列，每项显示角标数字。当前选中项高亮（`color: #8B5A2B`，加粗）。
- **列表区**：显示当前状态 Tab 下的所有任务，按项目分组。选中任务高亮边框（`#d4a574`）。
- **详情区**：选中任务后右侧显示详情，未选中时为空状态提示。

### 导航项（6 个）

1. 待启动 — 角标显示待启动任务数
2. 进行中 — 角标显示运行中任务数
3. 待决策 — 角标显示待决策数（红色高亮角标）
4. 待验收 — 角标显示待验收数
5. 已完成 — 无角标
6. 项目 — 切换到项目管理视图

**跨项目视图：** 状态 Tab（前 5 个）不按项目筛选，展示所有项目的该状态任务，按项目名分组。每组上方显示项目名作为分组标题。

### 项目管理视图

点击"项目"导航项时，列表区显示项目列表。每项显示：
- 项目名
- 工作目录路径
- 任务状态标签汇总（如"2进行中 · 3待启动"）
- 右上角"+ 新建"按钮创建项目
- 长按/右键删除项目

### 详情区（右侧面板）

选中任务后展示，包含：
- 标题 + 项目名 + 优先级
- 任务描述
- 运行状态（轮次、费用、会话状态）
- 操作按钮（根据状态动态变化）：
  - 待启动：开始、编辑、删除
  - 进行中：停止、继续
  - 待决策：显示待决策提示
  - 待验收：批准、拒绝（带反馈输入）
  - 已完成：仅查看
- 日志预览（最近 N 行，等宽字体，暖色背景 `#fdf8f0`）

### 弹窗（Modal）

以下操作用居中 Modal 展示：
- 新建任务：标题 + 描述 + 优先级 + 最大轮次 + "创建为待启动" / "创建并启动"
- 编辑任务：标题 + 描述
- 新建项目：名称 + 工作目录（带目录选择器）
- 决策对话框：问题 + 选项按钮 + 自定义回答
- 日志查看器：全屏滚动日志（深色终端风格改为暖色风格）
- 审核对话框：摘要 + 批准/拒绝按钮

```
├── CreateProjectModal

```

## 数据流（不变）

- REST API：`fetch` → `localhost:14567/api/*`，复用现有路由
- WebSocket：`ws://localhost:14567/ws`，复用现有消息类型
- 任务生命周期状态机不变：pending → running → deciding/reviewing → done

## 状态标签映射

```
pending    → 待启动
running    → 进行中
deciding   → 待决策
reviewing  → 待验收
done       → 已完成
```

## 保留项

- 服务端所有路由、数据库、会话管理、钩子系统 —— 全部复用
- WebSocket 消息类型不变
- 共享类型定义不变（仅 UI 标签文本映射）

## 移除项

- 桌面端 5 列看板布局（KanbanBoard）
- 项目列表页旧组件（ProjectList 的 mini-kanban 布局）
- 移动端适配代码（本项目不涉及）
